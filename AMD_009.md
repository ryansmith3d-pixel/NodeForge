### AMD-009 — Public GitHub Repository
Affects: All phases  
Status: COMPLETE  
Decided: 2026-03  
Reason: The project needs to be externally visible to serve its strategic purpose — the
thesis is not credible as a private folder. A public repo is also the minimum artifact
for any reviewer evaluating the project. Going public at Phase 5 complete is the right
moment: the architecture is clean, the decisions are documented, and the test suite
demonstrates reliability. An in-progress repo is not a liability; it is evidence of
active development.  
Change: Project hosted at https://github.com/YOUR_USERNAME/NodeForge (public).
Repository structure:
- `src/nodeforge/` — package source (unchanged)
- `tests/` — test suite (unchanged)
- `docs/` — blueprint, amendments, session workflow, all phase summaries
- `README.md` — thesis-first public framing
- `.gitignore` — Python/Windows standard exclusions
- `nodeforge.toml` — committed as a config example (no secrets)

The README leads with the thesis argument, not the tech stack. A reviewer landing on
the repo should understand why the project exists before they understand what it does.

Phase summary documents live in `docs/` and are committed alongside code. They serve
as both a project record and a public demonstration that architectural decisions were
intentional and traceable.

Done when: Repo is live, all existing phase artifacts are committed, and the README
renders correctly on GitHub.
