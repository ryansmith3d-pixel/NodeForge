## Architecture Established

**Token file is open and dynamic.** New roles are added by editing the JSON. The Qt tool
regenerates its row list on load — no hardcoded widget list. This prevents the schema from
becoming a maintenance liability as demo complexity grows.

**Animation separation.** The token file provides color endpoints (e.g. `running`,
`running_highlight`). Pulse/transition logic belongs to the consumer (D3/CSS), not the token
layer. This keeps the schema declarative and the animation logic where it can be iterated without
touching the token contract.

**Broadcast model.** Every token update sends the full token object. Receivers replace state
wholesale. This is simpler than diffing and consistent with the determinism thesis — the state
at any moment is fully described by the most recent broadcast.

## Seed Token File

Agreed minimal set — placeholder hex values, to be replaced via the tool:

```json
{
  "surface": {
    "canvas": "#1a1a1f",
    "panel": "#24242c"
  },
  "node": {
    "default": "#2e2e3a",
    "selected": "#7eb8f7"
  },
  "node_status": {
    "pending": "#555568",
    "running": "#f7c948",
    "running_highlight": "#ffe89a",
    "complete": "#4ab88a",
    "failed": "#c0392b"
  },
  "edge": {
    "default": "#3a3a4a",
    "selected": "#7eb8f7"
  },
  "semantic": {
    "alert": "#f7c948",
    "error": "#c0392b"
  }
}
```

Per-view edge encoding and node status encoding confirmed as requirements. Specific per-view
roles deferred — will be added to the registry as demo views are built.

## Qt Scaffold Plan

Three components:

- `tokens.seed.json` — seed file above, in `tools/color-designer/`
- `token_store.py` — loads JSON, flattens to dot-notation dict, saves back. No UI.
- `main.py` — QMainWindow with scrollable dynamic row list. Each row: label, swatch, hex
  field, picker button. Load / Save in toolbar. Rows generated from token file keys on load.

## Competitive Landscape Note (carried from April 8)

Per-view edge encoding decision is consistent with the thesis counter-position established in
the April 8 session: every color encodes a named, inspectable semantic role. Per-view encoding
makes the projection explicit — the same edge may carry different visual meaning in different
views, and that difference is declared, not implicit.
