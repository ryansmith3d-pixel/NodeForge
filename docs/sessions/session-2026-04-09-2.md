## Architecture Established

**The node graph is the source of truth.** Token file and SSE broadcast are outputs.
Schema is an input constraint. Nothing hardcoded — node types, token roles, and view
representations are all data-driven.

**Token file is open and dynamic.** New roles added by editing JSON. Schema node
regenerates port list on reload. This prevents the schema from becoming a maintenance
liability as demo complexity grows.

**Animation separation.** Token file provides color endpoints (e.g. `running`,
`running_highlight`). Pulse/transition logic belongs to D3/CSS layer, not token layer.

**Broadcast model.** Full token object sent on every update. No diffs. Receivers replace
state wholesale. Consistent with determinism thesis.

**The node graph UI is an argument.** Color Designer is a node graph. Idiograph is a
node graph. Two different problem domains, same architecture. This is not a coincidence
— it is structural evidence that the architecture generalizes.

## Node Graph UI — Significance

Diagrams produced by Ryan during session established:

1. **Pipeline model** (Diagram 1) — Source → Generate → Assign → Write/Drive. Schema is
   the assignment target. Three input paths: Color Swatch, Color Array, Read from file.

2. **Cross-app highlight** (Diagram 2) — Selecting a token role in Color Designer
   highlights the corresponding UI element in the live Idiograph preview. Two SSE event
   types: `token.update` (value changed) and `token.focus` (role selected). Deferred.

3. **Color Array internal rows** (Diagram 4) — "+ New Item" appends rows inside the node
   body. Not external port connections. Convenience feature.

4. **Node view switching** (Diagram 6) — Button strip at node bottom edge cycles through
   view representations. Same data, different projection. The multi-viewer model applied
   at node level.
