## What Was Built

### `src/idiograph/domains/color_designer/`

| File | Contents |
|---|---|
| `__init__.py` | `register_color_designer_handlers()` — lazy import pattern, matches arXiv convention |
| `handlers.py` | Six handler implementations: `color_swatch`, `color_array`, `schema`, `assign`, `array_assign`, `write_tokens` |
| `pipeline.py` | `COLOR_DESIGNER_PIPELINE` — canonical ArrayAssign pipeline Graph |

### Handler Summary

| Handler | params | returns |
|---|---|---|
| `color_swatch` | `hex`, `label` | `{color, label}` |
| `color_array` | `colors: [{label, hex}]` | `{color_array}` |
| `schema` | `token_file` | `{token_dict}` |
| `assign` | `role` | `{assignment: {role, hex}}` |
| `array_assign` | *(none)* | `{assignments: [{role, hex}]}` |
| `write_tokens` | `token_file` | `{status, count, path}` |

Input resolution for `array_assign` and `write_tokens`: content-shape inspection
(inspect `inputs.values()` for key presence). Closed vocabulary, no edge labels needed.

### Qt Node Changes

`to_idiograph_node() -> Node` added to all six Qt node classes. Each returns an
Idiograph `Node` using `self.node_id` (AMD-018) as the graph node ID.

| Qt class | Idiograph type |
|---|---|
| `SwatchNode` | `color_swatch` |
| `ArrayNode` | `color_array` |
| `SchemaNode` | `schema` |
| `AssignNode` | `assign` |
| `ArrayAssignNode` | `array_assign` |
| `WriteNode` | `write_tokens` |

### Canvas

`build_graph() -> Graph` added to `canvas.py`. Walks scene items for `BaseNode`
instances, calls `to_idiograph_node()` on each, walks live wires for edges.
Returns a complete Idiograph `Graph` — topology from wires, params from nodes.

### main.py

- `register_color_designer_handlers()` called at boot in `MainWindow.__init__`
- Save button wired to `execute_graph()` via `asyncio.run(execute_graph(canvas.build_graph()))`
- Qt canvas is now a view — it collects state and hands off to the executor

## What This Means

The color designer pipeline runs through the same executor as the arXiv pipeline.
Domain-agnosticism is no longer a claim — it is demonstrated by two running domains.
The Qt canvas is a view into graph state, not an execution engine. The thesis holds.
