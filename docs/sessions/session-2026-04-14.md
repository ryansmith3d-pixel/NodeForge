### Pass 3 — Coverage analysis

**Verdict: GREEN**

| Metric | Overall | Recent | Mid | Early |
|---|---|---|---|---|
| % with counts_by_year_len ≥ 3 | 100% | 100% | 100% | 100% |
| % with counts_by_year_len ≥ 1 | 100% | 100% | 100% | 100% |
| % with counts_by_year_len = 0 | 0% | 0% | 0% | 0% |
| Median counts_by_year_len | — | 5 | 9 | 14 |

Full α/β ranking is viable. `fallback_parameters: null`. No age filter required.

The median scaling (Recent=5, Mid=9, Early=14) is the expected shape — older papers
have had more years to accumulate year-over-year data. Crucially, the Recent band
(where citation lag is the known limitation) still has median 5 points. Acceleration
computation is viable even in the most challenging band.

The two known limitations offset each other: citation lag is worst for recent papers,
but counts_by_year coverage is also worst for recent papers — meaning the β term is
naturally weakest where the lag caveat is most applicable. The system is self-consistent.

## Key Decisions

| # | Decision | Rationale |
|---|---|---|
| 1 | Stratified sampling by year band, not random | Random sample would average away systematic coverage gaps at the tails |
| 2 | 50 papers per seed (100 total) | Sufficient for coverage distribution; well inside API budget |
| 3 | Sort by cited_by_count:desc within each band | Conservative — if high-citation papers have thin coverage, β is in serious trouble |
| 4 | `cites:` filter, not `cited_by:` | Spec correction mid-spike; `cited_by` is a field, not a filter key |
| 5 | Merge commit (not squash) | Preserves per-pass commit history as audit trail |

## Spec Correction

**`cited_by:` → `cites:`** in the Pass 2 section of the spike spec.

OpenAlex does not have a `cited_by:` filter. The correct filter for "papers that cite
work X" is `cites:<openalex_id>`. Claude Code caught this during Pass 2 execution,
surfaced it per the anti-drift protocol (flag, don't fix silently), and it was
corrected in a dedicated commit (`acf049e`) before the spec froze.

Process note: this is the anti-drift pattern working correctly. The spec was wrong;
the code was right; the human resolved it explicitly.
