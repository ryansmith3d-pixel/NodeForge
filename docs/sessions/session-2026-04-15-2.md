### PaperRecord model

`src/idiograph/domains/arxiv/models.py` — created new.

Typed Pydantic model representing a single paper as it flows through the
pipeline. All fields declared with `Field(description=...)` per AMD-001.

Fields:
- Identity: `node_id`, `arxiv_id`, `doi`, `openalex_id`
- Metadata: `title`, `year`, `authors`, `abstract`, `citation_count`
- Traversal provenance: `hop_depth`, `root_ids`
- Downstream pipeline fields (default None, populated by later nodes):
  `community_id`, `pagerank`, `topological_depth`,
  `relationship_type`, `semantic_confidence`

Module-level helpers:
- `make_node_id(work: dict) -> str` — canonical node_id derivation.
  Priority: arxiv_id > doi > openalex_id.
- `reconstruct_abstract(inverted_index: dict | None) -> str | None` —
  converts OpenAlex compressed abstract format to plain text.

### Node 0 — `fetch_seeds()`

`src/idiograph/domains/arxiv/pipeline.py`

Accepts `list[dict]` of seed identifiers (each `{"arxiv_id": ...}` or
`{"doi": ...}`). Resolves each against OpenAlex. Returns a tuple:
`(resolved: list[PaperRecord], failures: list[dict])`.

Behavior:
- Resolves each seed independently via OpenAlex works endpoint with
  identifier filter.
- Sleeps 150ms between API calls (after first call).
- On individual resolution failure: logs warning, appends to failures,
  continues.
- Raises `ValueError` if seed_ids is empty, or if all seeds fail.
- Seed nodes carry `hop_depth=0`, `root_ids=[node_id]` (AMD-017).
- Fields fetched via `select=`: `id,ids,title,publication_year,
  authorships,abstract_inverted_index,cited_by_count`.
- `_get_api_key()` loader in module — callers pass `api_key` explicitly
  so functions are testable without environment variables.

### Node 3 — `backward_traverse()`

`src/idiograph/domains/arxiv/pipeline.py`

BFS backward traversal from all resolved seed nodes, depth=1 and depth=2.

Behavior:
- Re-fetches seeds with `referenced_works` included (Node 0's select is
  metadata-only — a separate batch fetch at the start of traversal is
  the correct boundary-preserving approach).
- Fetches references in batches of 50 IDs per OpenAlex call
  (pipe-separated filter). Sleeps 150ms between batch calls.
- Deduplicates by `node_id`. On collision: keeps lowest `hop_depth`,
  merges `root_ids` (sorted union — deterministic, testable).
- Excludes seed nodes from traversal output (seeds are already graph
  roots). Covers seeds-citing-seeds case explicitly.
- Papers with no OpenAlex record: silently skipped, logged at DEBUG.

Scoring function (`_node3_score()`):
```
score = citation_count × log(hop_depth + 1) × (1 / recency_weight)
recency_weight = e^(years_since_publication × lambda_decay)
```
- `years_since_publication`: current year − paper.year.
  If `paper.year` is None: treated as 0 (no penalty for missing data).
- Natural log (`math.log`).
- Papers sorted descending by score, capped at `n_backward`.

## Implementation Decisions

| # | Decision | Rationale |
|---|---|---|
| 1 | `asyncio.run()` in tests, not `pytest-asyncio` | No new dependencies, no config churn. Real async fixture chaining not needed at this stage. Revisit if orchestrator layer requires it. |
| 2 | Node 3 re-fetches seeds for `referenced_works` | Node 0's contract is metadata-only (no `referenced_works` in select). Widening Node 0's select would violate its single-responsibility boundary. Extra round trip is bounded and paid once. |
| 3 | `root_ids` sorted on merge | Union has no defined order. Sorted is deterministic and testable. Insertion-order would be an implicit contract on traversal order. |
| 4 | Seeds excluded from traversal output | Seeds are root nodes, not traversal results. Covers seeds-citing-seeds edge case. Explicit test coverage added. |
