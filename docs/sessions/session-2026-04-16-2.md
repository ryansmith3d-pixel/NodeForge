### Node 4.5 — `clean_cycles()`

`src/idiograph/domains/arxiv/pipeline.py` — pure function, no I/O, no network, no async,
no mutation of inputs. Added after `forward_traverse` and before the module-level
`ARXIV_PIPELINE` constant to keep Node 0 → 3 → 4 → 4.5 contiguous.

**Signature:**

```python
def clean_cycles(
    nodes: list[PaperRecord],
    edges: list[CitationEdge],
) -> CycleCleanResult:
```

**Algorithm (implemented verbatim from spec):**

1. Build `networkx.DiGraph` from `(nodes, edges)`.
2. Loop `nx.find_cycle(G, orientation="original")` until `NetworkXNoCycle`.
3. For each cycle, score each member edge as
   `citation_count[source] + citation_count[target]`.
4. Select weakest by `min(edges, key=lambda e: (citation_sum, source_id, target_id))`
   — single-tuple sort gives deterministic lex tiebreaker.
5. Remove from graph and edge list; append `SuppressedEdge` record.
6. Return `CycleCleanResult(cleaned_edges, cycle_log)`.

**Safety bounds:**

- Iteration cap = `len(edges)`. Exceeding raises `RuntimeError` (bug signal, not
  malformed input).
- Missing-node citation lookups treated as `citation_count=0`, WARNING logged once per
  unique missing `node_id` (dedup via `warned_missing` set — implementation refinement
  over the spec's silence on frequency).

**Note on `nx.find_cycle` tuple shape:** `orientation="original"` returns 3-tuples
`(u, v, "forward")`, not 2-tuples. Implementation indexes `edge[0]/edge[1]` — tuple-size-
agnostic. The spec gets a note added post-session.

### Data models

`src/idiograph/domains/arxiv/models.py` — three models added after `CitationEdge`,
before the `make_node_id` helper:

- `SuppressedEdge` — source_id, target_id, citation_sum (weight at removal),
  cycle_members (node_ids in the cycle being broken, in traversal order).
- `CycleLog` — suppressed_edges list, cycles_detected_count, iterations, and
  `affected_node_ids` as a derived `@property` (union of source/target across
  suppressed edges). Derived, not stored — no drift possible. Node 6 calls this when
  deciding which nodes get `topological_depth: null`.
- `CycleCleanResult` — `cleaned_edges`, `cycle_log`. Named access at call sites over
  tuple unpacking; extensible without refactor.

## Implementation Decisions

| # | Decision | Rationale |
|---|---|---|
| 1 | `CycleCleanResult` as Pydantic model, not tuple | Named access at call sites; extensible without refactor. Matches `PaperRecord` precedent. |
| 2 | `affected_node_ids` as `@property`, not stored field | Derived from `suppressed_edges` — storing would permit drift. Single source of truth. |
| 3 | Missing-node WARNING deduped per node_id | Spec silent on frequency; log flooding on repeat lookups is real. Dedup via `warned_missing: set`. |
| 4 | `min(..., key=(citation_sum, source_id, target_id))` | Single tuple sort gives score-then-lex ordering; no separate sort pass. |
| 5 | Iteration cap = `len(edges)` | Cannot require more passes than there are edges; exceeding signals a loop bug. |
| 6 | `clean_cycles` placed before `ARXIV_PIPELINE` constant | Keeps Node 0 → 3 → 4 → 4.5 contiguous; executor constant stays at module bottom. |
| 7 | New models placed after `CitationEdge`, before `make_node_id` | Groups Pydantic models contiguously; helper stays at module bottom. |
| 8 | Pure function, no mutation of inputs | Matches Nodes 3/4 pattern. Testable, re-runnable with identical results. |

## Spec Gaps Surfaced (Post-Freeze Fix List)

1. **`CitationEdge` was never defined before Node 4.5.** The frozen pipeline spec
   defines its *schema contract*; no prior node had introduced the model. Surfaced by
   Claude Code before any code was written, resolved by placing the model in this
   commit. Future node specs should audit prerequisite models before writing.

2. **`cycles_detected_count` vs `len(suppressed_edges)`.** Under the one-cycle-per-
   iteration algorithm in the spec, these are always equal. The spec's "may exceed"
   language presumed an `nx.simple_cycles` enumeration variant that was considered but
   not specified. `test_nested_cycles` `>=` assertion is correct but trivially so.
   Post-freeze: keep the field (semantic value holds, stays correct under future batch-
   enumeration variants), tighten wording.

3. **`nx.find_cycle` tuple shape under `orientation="original"`.** Returns 3-tuples,
   not 2-tuples. Spec did not note this. Implementation is defensive (tuple-size-
   agnostic indexing). Spec gets a note.

4. **Spec header referenced `session-2026-04-16.md` as a companion document at freeze.**
   That session doc predates this spec's creation. Corrected before implementation to
   list only `spec-arxiv-pipeline-final.md` (frozen v3). Pattern rule: spec headers
   reference frozen prior artifacts only.
