### Node 4 — `forward_traverse()`

`src/idiograph/domains/arxiv/pipeline.py` — three private helpers and the main function
added after the Node 3 block. No existing functions modified.

**Private helpers:**

`_compute_velocity(cited_by_count, pub_year, current_year) -> float`
- `cited_by_count / max(1, (current_year - pub_year) * 12)`
- Returns 0.0 if `pub_year` is None

`_compute_acceleration(counts_by_year, acceleration_method) -> float | None`
- Sorts `counts_by_year` ascending before computation
- Requires ≥3 entries; returns None if fewer (triggers per-paper β=0 fallback)
- `"first_difference"`: mean of year-over-year velocity deltas
- `"regression"`: raises `NotImplementedError` — slot reserved, not stubbed
- Any other value: raises `ValueError` — defensive guard

`_node4_score(velocity, acceleration, pub_year, current_year, alpha, beta, lambda_decay) -> float`
- `score = alpha * velocity + beta * acceleration * recency_weight`
- `recency_weight = math.exp(years * lambda_decay)` — rewarded, not penalized
- If `acceleration` is None: `beta` treated as 0 for that paper only

**`forward_traverse()` function:**

```python
async def forward_traverse(
    seeds: list[PaperRecord],
    api_key: str,
    n_forward: int,
    alpha: float,
    beta: float,
    lambda_decay: float,
    acceleration_method: str = "first_difference",
    current_year: int | None = None,
) -> list[PaperRecord]:
```

- `current_year` defaults to `datetime.date.today().year` if None
- Fetches citing papers per seed via OpenAlex filter `cites:<openalex_id>`,
  `per-page=200` (OpenAlex max — one call per seed, no pagination loop)
- `select=` includes `counts_by_year` — not available from Node 0 or Node 3 fetches
- 150ms sleep between API calls (after first), matching project standard
- `counts_by_year` held in a parallel `counts_by_id: dict[str, list[dict]]` during
  traversal — `PaperRecord` has no field for it and cannot be widened per spec
- Deduplication on `node_id`: keep lowest `hop_depth` (all forward papers are
  `hop_depth=1`), merge `root_ids` as `sorted(set(...) | {seed.node_id})` — mirrors
  AMD-017 behavior in `backward_traverse`
- Seed nodes excluded from results
- Papers with no OpenAlex record: skipped, logged at DEBUG
- Sorted descending by score, capped at `n_forward`

## Implementation Decisions

| # | Decision | Rationale |
|---|---|---|
| 1 | `counts_by_id` parallel dict, not PaperRecord field | PaperRecord cannot be widened per spec; `counts_by_year` is computation-time input, not pipeline output — not required in renderer data contract |
| 2 | `per-page=200` on cites: query | OpenAlex max; one call per seed without pagination loop matches spec intent |
| 3 | `_compute_acceleration` checks `"regression"` before point count | `NotImplementedError` fires regardless of data size — test 9 expectation met cleanly |
| 4 | `ValueError` for unknown acceleration_method | Defensive guard; default is `"first_difference"`, `"regression"` is reserved — anything else is a caller error |
| 5 | ruff format applied to new code only | Formatting pre-existing code in pipeline.py would reflow existing functions, violating the "do not modify existing functions" constraint. ruff check passes clean. |

## Prompt Workflow Notes

Two workflow improvements formalized this session:

**Prompt files over in-chat pastes.** The Node 4 prompt was produced as
`tmp/node4-prompt.md` with a companion spec at `docs/specs/spec-node4-forward-traverse.md`.
Claude Code reads both before writing code. This prevents paste truncation and gives
Claude Code a re-readable anti-drift reference throughout implementation.

**Future prompt constraint fix:** the ruff constraint should read *"ruff format your new
code; do not reformat pre-existing code"* — not "ruff format the whole file." The
current wording creates a conflict on files with pre-existing style variance.

**Test count in prompts:** prompt predicted 68 + 9 = 77; actual baseline on main was 72,
yielding 81. Future prompts should pull the live count from the repo (`uv run pytest
tests/ --co -q | tail -1`) rather than carrying forward a number from a session doc.

commit and push, not reconstructed afterward. Workflow: Claude Code finishes and reports
→ report pasted here → summary file produced → dropped in repo → included in final commit.
