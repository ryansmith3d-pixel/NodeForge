Phase 8) with a baseline-relative rule:

Rationale: any hardcoded count rots on the next node. The invariant — never regress,
always add — is what actually matters and is durable across phases. Makes the "record

### docs/specs/spec-node4.5-cycle-cleaning.md — two post-freeze clarifications

**Fix 1: `cycles_detected_count` wording.** Original description claimed the count
"may exceed `len(suppressed_edges)` when one removal breaks multiple cycles." Under
the one-cycle-per-iteration algorithm actually implemented via `nx.find_cycle`, that
inequality is never reached — each iteration finds exactly one cycle and suppresses
exactly one edge. The "may exceed" framing referenced an `nx.simple_cycles`
enumeration variant that was considered during design but not specified.

New description retains the field (forward-compatible with a batch-enumeration
variant where one removal *could* resolve multiple detected cycles) and is explicit
about what it currently measures under the implemented algorithm.

**Fix 2: `nx.find_cycle` tuple shape.** Spec's algorithm section treated `cycle_edges`
as `(u, v)` pairs. Under `orientation="original"`, `nx.find_cycle` returns 3-tuples
`(u, v, "forward")`. The implementation already indexes positionally ([0], [1]) so
the code is tuple-size-agnostic, but the spec didn't say so. Added an inline comment
warning not to destructure as `(u, v)`.

## Workflow Observations

**Codecov blocked the merge on green CI.** Branch protection on `main` requires
coverage signal even though this PR touched no code logic. Admin-merge was the right
call for a pure doc/config PR — no behavioral risk to justify blocking on a coverage
metric that structurally can't move. Worth noting: if non-code housekeeping PRs stack
up, a "skip coverage check for docs-only" carveout in branch protection would remove
the friction without weakening the rule for code PRs.

**Push surfaced a GCM HttpRequestException header that wasn't a failure.** Objects
uploaded, branch tracking confirmed, PR URL printed. Noise from the credential helper
on Windows Git — not actionable, worth recognizing on sight next time so the session
doesn't pause on it.

**`uv run ruff` vs `uv tool run ruff`.** `ruff` isn't in the project venv, so `uv run`
can't resolve it. `uv tool run ruff` installs on demand and runs in isolation — this
is the right invocation for lint-only checks. Worth documenting in CLAUDE.md's
environment section next time that file gets touched.
